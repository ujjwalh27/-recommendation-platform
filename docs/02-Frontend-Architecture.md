# Frontend Architecture

## Overview

The frontend of the Recommendation Platform is built using **React** and **Vite**. It follows a **feature-based architecture**, where related components and logic are grouped together instead of organizing files only by type.

The frontend is responsible for:

- Displaying the video feed
- Playing videos
- Tracking user behaviour
- Managing watch sessions
- Displaying live analytics
- Communicating with the backend (future)

# Feature-Based Architecture

Instead of organizing by file type:

```
components/

pages/

hooks/

utils/
```

we organize by business feature.

Example:

```
feed/

player/

analytics/

session/
```

Each feature owns its own components and business logic.

Advantages:

- Easier maintenance
- Better scalability
- Cleaner imports
- Easier onboarding for developers

# Feature Responsibilities

## Home

Responsible for displaying the application's home page.

---

## VideoFeed

Responsibilities:

- Display recommendation feed
- Render video cards
- Future infinite scrolling
- Feed refresh

---

## VideoCard

Responsible for:

- Video container
- Video player
- Metadata
- User actions

---

## VideoPlayer

Responsible for:

- HTML5 video playback
- Video events
- Behaviour tracking

Tracked Events:

- Loaded
- Play
- Pause
- Seek
- Watch Progress
- End

---

## BehaviourTracker

Responsible for collecting all behavioural events generated by the user.

Responsibilities:

- Store events
- Notify listeners
- Support analytics

---

## WatchSessionManager

Responsible for managing one watch session.

Responsibilities:

- Start session
- Update session
- End session
- Maintain statistics

---

## AnalyticsPanel

Responsible for displaying:

- Behaviour events
- Session summary
- Live analytics
