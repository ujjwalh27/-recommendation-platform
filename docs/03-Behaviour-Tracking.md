# Behaviour Tracking

## Overview

Behaviour Tracking is the foundation of the Recommendation Platform.

Every interaction performed by the user while watching videos is captured as a behaviour event. These events are later analyzed to understand user preferences, calculate engagement, and generate personalized recommendations.

Without behaviour tracking, the recommendation engine cannot determine what content the user enjoys.

---

# Objectives

The Behaviour Tracking module is responsible for:

- Capturing user interactions
- Recording watch progress
- Measuring engagement
- Updating watch sessions
- Providing input to the recommendation engine
- Supporting future analytics dashboards

# Current Events

The application currently captures the following events.

## LOADED

Triggered when the video metadata has been loaded.

Purpose:

- Determine video duration
- Prepare playback session

Example:

```
{
    event: "LOADED",
    duration: 42.3
}
```

---

## PLAY

Triggered when the user starts or resumes playback.

Purpose:

- Start watch session
- Increase play count

Example:

```
{
    event: "PLAY",
    currentTime: 0
}
```

---

## PAUSE

Triggered when playback is paused.

Purpose:

- Measure interruptions
- Update session statistics

Example:

```
{
    event: "PAUSE",
    currentTime: 18.6
}
```

---

## WATCH_PROGRESS

Triggered continuously while the video is playing.

Purpose:

- Track watch duration
- Calculate completion percentage

Example:

```
{
    event: "WATCH_PROGRESS",
    currentTime: 21.4,
    percentage: 48.7
}
```

---

## SEEKING

Triggered when the user starts moving the playback position.

Purpose:

- Detect skipping behaviour

---

## SEEKED

Triggered after seeking completes.

Purpose:

- Determine where the user jumped

---

## ENDED

Triggered when the video finishes.

Purpose:

- Close watch session
- Calculate completion
- Update recommendation signals

---

# BehaviourTracker

The BehaviourTracker acts as the event collection layer.

Responsibilities:

- Record every event
- Store events in memory
- Notify subscribers
- Provide data for analytics

Current implementation:

```
VideoPlayer

↓

BehaviourTracker.record()

↓

AnalyticsPanel

↓

WatchSessionManager
```

---

# Event Structure

Every event follows a common structure.

```
{
    event,
    currentTime,
    percentage,
    time
}
```

Meaning:

| Field | Description |
|--------|-------------|
| event | Behaviour type |
| currentTime | Playback position |
| percentage | Completion percentage |
| time | Event timestamp |

# Why Behaviour Tracking Matters

Behaviour events allow the system to answer questions such as:

- Which categories does the user watch most?
- Which videos are skipped?
- Which videos are replayed?
- Which videos are completed?
- Which creators receive more engagement?

These insights directly influence recommendation quality.

# Behaviour Scoring

Each behaviour contributes differently toward user interest.

Example:

| Behaviour | Impact |
|------------|---------|
| Video Completed | Very High |
| Replay | High |
| Like | High |
| Save | Very High |
| Share | Very High |
| Pause | Neutral |
| Seek | Medium |
| Skip | Negative |

These weights will later be used by the Recommendation Engine.
